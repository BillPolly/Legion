    Hi everyone and thanks for sharing the resources!

    I’ve read Asynchronous Programming in MicroPython by Peter Hinch which was quite good, specially if you're into embedded systems (as I was in the past 😅). I also found Python Asyncio: The Complete Guide by Jason Brownlee particularly inspiring, it actually inspired me to build my own guide. Other links were also very interesting for me.

    I put together asynchronous-python, a comprehensive and hands-on guide that covers asynchronous programming in Python from core concepts to practical use cases. It’s compatible with the latest Python versions (3.11 to 3.13) and includes working code snippets with passing unit tests.

    Some of the topics covered include:

    • Event loops, coroutines, and asyncio APIs
    • CPU-bound vs I/O-bound workloads
    • Process creation (fork vs spawn)
    • Synchronization primitives like locks and semaphores
    • asyncio.Queue, Future, callbacks, and error handling
    • Async-related improvements in Python 3.11 to 3.13
    • ASGI vs WSGI, and building async web APIs with Starlette
    • Deep dives into AnyIO and Trio with structured concurrency
    • Task scheduling with Celery and APScheduler

    Would appreciate any feedback or comments 😊

    That looks good as a guide to CPython's asyncio. From a MicroPython perspective I have a couple of reservations.

    Firstly it includes features absent in MicroPython's lightweight asyncio implementation: this could be confusing to beginners. Some of the examples require "big iron": for example trying to launch 10,000 tasks on a microcontroller is likely to result in tears before bedtime...

    peterhinch May 17, 2025

    -

    I'm not sure what to recommend. When I first approached MicroPython asyncio I studied the CPython docs. Despite long experience of cooperative scheduling, I found the docs confusing, mainly because asyncio had grown by accretion over numerous iterations. I set about writing the tutorial to describe only the MicroPython subset; also to cover firmware-specific issues.

    You rightly cover features beyond the MP subset, including OS API elements unavailable to a baremetal port (e.g. fork and spawn). A lot of the tools you describe are beyond the scope of microcontrollers. While this is of great interest, it might confuse a beginner in microcontroller coding.

    Your best bet might be a discussion of the MP subset with pointers to the docs.
