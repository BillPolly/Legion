CNL vs DSL Comparison

Controlled Natural Language makes declarative components more accessible

📊 Counter Component

CNL (Natural Language)

Readable, intuitive syntax

Define Counter with state:
  Create a container with class "counter" containing:
    A heading showing the count
    A button labeled "+" that increments the count on click
    A button labeled "-" that decrements the count on click
    A button labeled "Reset" that sets the count to 0 on click

DSL (Domain Specific Language)

Concise, symbolic syntax

Counter :: state =>
  div.counter [
    h2 { state.count }
    button @click="state.count++" { "+" }
    button @click="state.count--" { "-" }
    button @click="state.count = 0" { "Reset" }
  ]

🎯 Conditional Rendering

CNL (Natural Language)

Natural conditional expressions

Define Menu with data:
  A button that toggles menuOpen on click:
    "Menu" + (if menuOpen then "▲" else "▼")
  
  When menuOpen is true:
    A container with class "dropdown" containing:
      A menu item labeled "Save"
      A menu item labeled "Open"
      A menu item labeled "Exit"

DSL (Domain Specific Language)

Compact conditional syntax

Menu :: data =>
  div [
    button @click="data.menuOpen = !data.menuOpen" { 
      "Menu " + (data.menuOpen ? "▲" : "▼")
    }
    
    if (data.menuOpen) [
      div.dropdown [
        div.menu-item { "Save" }
        div.menu-item { "Open" }
        div.menu-item { "Exit" }
      ]
    ]
  ]

📋 List Iteration

CNL (Natural Language)

English-like iteration

Define TodoList with data:
  A heading showing "My Tasks"
  
  For each task in the tasks:
    A container with class "task" containing:
      A checkbox bound to the task's completed
      A span showing the task's title
      A button labeled "×" that removes the task

DSL (Domain Specific Language)

Functional iteration syntax

TodoList :: data =>
  div [
    h2 { "My Tasks" }
    
    for task in data.tasks [
      div.task [
        input[type=checkbox] @bind="task.completed"
        span { task.title }
        button @click="removeTask(task)" { "×" }
      ]
    ]
  ]

CNL Language Features

🎯 Natural Component Definition

Define components using plain English: "Define ComponentName with data:"

📝 Intuitive Element Creation

Create elements naturally: "A heading showing the title"

🔗 Clear Data Binding

Express bindings clearly: "showing the user's name" or "bound to the value"

🎮 Simple Event Handling

Handle events naturally: "that increments the count on click"

🔄 Natural Conditionals

Use familiar words: "When condition is true:" or "If x then y else z"

🔁 Clear Iteration

Loop naturally: "For each item in the collection:"

Bidirectional Conversion

↔️ Two-Way Translation

Convert between CNL and DSL formats seamlessly

✅ Semantic Equivalence

Both forms express exactly the same component logic

📚 Progressive Learning

Start with CNL, gradually learn DSL syntax
